Tool-capable models

The chat tool sends function definitions on each turn and expects the model to emit tool_calls. Models without tool training (Gemma 1/2/3, Llama 2, base Mistral) ignore the tools and reply with text. Reliable picks:

  • qwen2.5 / qwen2.5:14b - reliable default
  • llama3.1 / llama3.2 - Meta, broad tool support
  • mistral-nemo, mistral-small
  • command-r-plus - heavy but very strong tool use
  • qwen3 / qwen3-coder

Verify any model with ollama show <model>; the Capabilities: line must include tools. Full catalog: ollama.com/search?c=tools.

Origin handling (no Ollama config required)

Ollama rejects chrome-extension:// origins by default. To save you from configuring OLLAMA_ORIGINS on every host, this extension automatically rewrites the outbound Origin header to http://localhost for requests to your saved Ollama URL. localhost is in Ollama's default allowlist, so chat turns just work.

The rewrite is scoped to the URL you save here; requests to Anthropic and other origins are not affected.

If 403s persist the rewrite may have failed to register (rare). Fall back to setting OLLAMA_ORIGINS="chrome-extension://*" on the host running Ollama and restart the server. macOS / Linux: OLLAMA_ORIGINS="chrome-extension://*" ollama serve. Windows: see docs/ask-ai-local-providers.md.

Ask AI is an in-plugin chat that queries your FortiMonitor environment via a curated set of v2 API tools. Pick the AI provider above (Anthropic, Ollama, or LM Studio); requires the matching credentials plus your FortiMonitor RW key. Shown by default; toggle off to hide the tile and provider settings.

Controls how many tools are sent to the AI on each chat turn. Bigger tier means a bigger prompt, more tokens per turn (and more cost when the provider is Anthropic), and more capability for the model to act on its own. Read-only is the safest default; promote to read+write only when you want the model able to call create/update/delete tools (still gated on you asking explicitly). The "all tools" tier loads the entire 260+ catalog and is for when you genuinely want maximum capability.

Local providers (Ollama, LM Studio) always get a curated handwritten-only subset regardless of tier - the 260+ codegen tools confuse small open models. Cloud Anthropic still gets the full catalog at the "all" tier.

Search Servers pages the full /server list and filters by a specific attribute (e.g., Model = FGT60F). Pick the attribute from your tenant's catalog, enter the value, exports matches as CSV. Read-only. Hidden by default.

SD-WAN Report crawls every monitored server's SNMP, agent, and network-service resources, classifies SD-WAN overlay / underlay / generic metrics, and exports CSV + JSON. Read-only. Hidden by default.

Best-Practice Assessment assesses the full instance against best-practice analyzers (incidents, users, instances, templates, monitoring policy workflow) and surfaces an 11-tab in-browser viewer with a single combined report download (or per-tab CSVs) for customer-facing delivery. Read-only. Hidden by default.

Pure generator; you save the actual config in each admin UI. Hidden by default.
