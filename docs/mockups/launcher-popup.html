Design decisions (approved 2026-04-17)

  • Width: 360px. Chrome popups can go up to 800px but 300–400 is the sweet spot for toolbar popups. 360 matches the density of existing mockups.
  • No subtitle. Title alone ("Unofficial FortiMonitor Toolkit"). Project is not affiliated with or a client of FortiCloud-the-product; subtitle removed to avoid implying otherwise.
  • Session-status strip. Every tool needs the FortiCloud cookie, so surfacing sign-in state up front prevents every tool from re-inventing its own error path. Clicking "Open login" / "Open console" opens https://fortimonitor.forticloud.com/ in a new tab.
  • Search always visible. Cheap at the current tool count; valuable as the list grows. No conditional render logic needed later.
  • Grouped sections. "Port scope", future groups could be "Monitoring", "Inventory", "Audit", etc. The tool-group-label is inline HTML so rearrangement is trivial.
  • Action-oriented badges. "Removes" (orange) and "Adds" (green). Operator reads intent, not risk-abstraction — matches how the tools are named and categorized.
  • No "Coming soon" entries. Do not advertise unshipped tools.
  • Click behavior. Click a tool card → chrome.tabs.create({url: '...'}) opens that tool's full-tab UI and the popup closes automatically. The existing five-step flow lives inside each tool's tab.
  • Disabled state (Variant B). When the XSRF-TOKEN cookie is missing, all tools are non-interactive and show "Sign in to FortiCloud to enable" in place of their description. User clicks the session-strip link to log in; popup can re-check on next open.
  • Icon/logo: placeholder "FM" tile. No custom glyph planned.
  • Footer. Version + a link to help/feedback. Can host a GitHub-issues link once the repo is public (FMN-41).

What's NOT in this mockup (intentionally)

  • Settings/options page — reserved for the ⋯ menu button; separate design pass.
  • Per-tool pinning/favoriting — scale-time feature; not needed at two tools.
  • Keyboard navigation affordances — handled in implementation (Tab, Enter, arrow keys); no visual mockup needed.
