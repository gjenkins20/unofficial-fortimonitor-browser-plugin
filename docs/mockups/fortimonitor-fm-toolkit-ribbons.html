FMN-86 — FM Toolkit attribution ribbon

Diagonal blue ribbon in the top-right corner of each visible UI feature added by the FM Toolkit, so the operator can tell at a glance which controls come from the extension vs. native FortiMonitor.

Live tuning — default ribbon (60×60)

Drag the sliders. Text indent shifts the words along the strip's diagonal axis: positive moves them toward the bottom-right of the strip (further into the visible corner), negative moves them toward the top-left (toward where the strip exits the host's top edge). Strip top / right reposition the whole strip, which also moves the text. Copy the values out of the CSS box at the bottom when it looks right.

3. Per-sub-header ribbons on the merged Instance cell — proposed update

  
Each FM Toolkit-added sub-header (IP Address, DNS Name, Type) gets its own xs ribbon. The native "Instance" sub-header is FortiMonitor's own column and stays unbadged. The merged-cell architecture is unchanged (DataTables forbids splitting these into sibling cells); the ribbons attach to the toolkit-owned <span> sub-headers inside the cell. Body rows render as today; only the sub-headers get ribbons.

  
Sub-headers in the live extension are ~18–22 px tall and ~110–200 px wide. The xs ribbon below uses those constraints (28×28 overlay, 5.5 px text) and renders the abbreviated text "FM TK" — at this scale the full "FM Toolkit" doesn't read. Default and compact variants keep the full "FM Toolkit" text.

4. Spec recap

  

    Visual — default: 60×60 px corner overlay; 90 px rotated 45° strip; top: 10.5px; right: -25px; text-indent: 5.5px;; 7.5 px / 600 text reading FM Toolkit. Compact: 40×40 px overlay; 60 px strip; top: 7px; right: -16.5px; text-indent: 3.5px;; 6 px text reading FM Toolkit. Xs (table sub-headers): 28×28 px overlay; 42 px strip; top: 5px; right: -10px; text-indent: 2.5px;; 5.5 px text reading FM TK (abbreviated — full text doesn't read at this scale). Background #1f6feb, white text, subtle drop shadow.

    Behavior: rendered once per FM Toolkit-added feature when its augmentation mounts; persists for the page lifetime; no animation; pointer-events: none so it never intercepts a click.

    Scope (in, today): each FM Toolkit-added sub-header (IP Address / DNS Name / Type) on /report/ListServers (FMN-71/75) — xs variant, attached to the .fmn-sub-hdr span. Native "Instance" sub-header stays unbadged. Helper-ready for future: any new visible UI element the extension adds — default or compact variant depending on host size.

    Scope (out): sidebar launcher (self-identifies as "FM Toolkit"), the extension popup (entirely the toolkit), behavioral-only augmentations like FMN-82.

    Settings: existing popup → ⚙ Settings checkbox stays; label changes to "Show 'FM Toolkit' badges on extension-added features", storage key renamed fm:showLoadedToast → fm:showFeatureBadges, default on.
