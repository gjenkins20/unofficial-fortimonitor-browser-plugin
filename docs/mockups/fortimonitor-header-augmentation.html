Click behavior (same for all variants)

What happens when the button is clicked
  • Button extracts server ID from location.pathname via regex /^\/report\/Instance\/(\d+)\/details/
  • Opens chrome.runtime.getURL('src/ui/app.html?tool=remove&serverIds={id}') in a new tab (does not replace the current FortiMonitor tab)
  • The plugin's Port Scope tool reads ?serverIds= on load (4-line change to app.js) and seeds the existing start.js textarea with that single ID
  • Operator lands on step 1 of the Remove Port Scope workflow with the instance already in the queue — they can add more, file-upload, or proceed directly

Remove vs Add: single button starts the Remove tool (primary FMN-36 pain point). Operator can flip to Add inside the tool. A future augmentation could be a split-button or second button in the same slot; deferred for now.
