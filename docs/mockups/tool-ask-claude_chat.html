Acceptance criteria

  • Header: purple AC icon badge, title, subtitle, Beta · Prototype badge, back-link to launcher.
  • Cost warning banner: always visible above the chat pane, --running info palette. Short, unambiguous.
  • Setup warning banner (State 2): --warn palette, appears only when keys are missing. Links to popup Settings.
  • User messages: right-aligned, --muted-soft background.
  • Assistant messages: left-aligned, card-style, streaming text appends live.
  • Tool-call cards: monospace font, sandy palette (--tool-bg), status pill top-right (running → done or error). Input pane opens by default while running; result pane expands on click when done.
  • Composer: textarea + Send / Stop / Clear chat. ⌘/Ctrl+Enter to send. Disabled while the loop is running; Stop button appears in its place.
  • Footer: tool count + attribution. Per-turn counter ("turn 1 · 2 tool calls") updates during runs.

State coverage

  • Empty (first open) ✓
  • Setup warning (keys missing) ✓
  • Mid-turn streaming with tool calls in flight ✓
  • Settled answer with tool-call history ✓
  • (Out of prototype scope — persisted history, multi-session recall, error recovery deep-dives.)

Transitional states worth noting for implementation

  • Text-block streaming: assistant bubble appears with an empty body, tokens append, closing end_turn removes any typing cursor.
  • Tool-call running → done: the status pill swaps in place; a result disclosure appends below input.
  • Tool-call error: pill turns red; an error detail disclosure appears. Loop continues unless stop_reason indicates otherwise.
  • Max-iterations ceiling: footer status reads max_iterations reached (8 turns); the user can ask a follow-up.
