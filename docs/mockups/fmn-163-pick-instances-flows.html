Diff vs current

  • + drop-zone + paste textarea as the lead affordance
  • + green format-hint card with example
  • + parse-result preview table (Name | Server ID), replaces chip list
  • + "Uses your active FortiMonitor session (cookies)" footer note
  • ~ omni-search retained, demoted under "or search by name / tag" divider

Open questions for operator

  • Keep chips somewhere (e.g., compact strip above the parse table for quick remove), or accept that the table row is the only display?
  • Show resolved-vs-unresolved warnings inline in the table (e.g., red row for "not in cache"), or in a .warn-list below?
  • Should "Load from current page selection" stay a button, or auto-trigger when the page-selection cookie is fresh?
