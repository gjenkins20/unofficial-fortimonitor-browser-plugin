  • Structure: injected as a <li class="pa-side-nav__top-level-item"> appended to the existing sidebar <ul>, so it participates in the same layout and doesn't break anything on collapse
  • Visual treatment: solid blue (#1f6feb) with white bold label and grid icon — same palette as the header-button mockup. Clearly reads as "this is not a native FortiMonitor feature"; no risk of someone filing a bug against Fortinet for something the plugin did
  • Persistence: re-injected by the FMN-69 framework on every SPA route change; keyed by data-fmn-entry="toolkit" so no duplicate injection on re-render

Click behavior (context-aware through the launcher, not the button)

  1. Click handler reads location.pathname at click time
  2. Opens chrome.runtime.getURL('src/ui/app.html?launcher=1&fromPath=<encoded path>') in a new tab
  3. Launcher parses fromPath and derives context:
    /report/Instance/42024060/details   →   { kind: "instance", serverId: "42024060" }
    /report/ListServers   →   { kind: "list" }
    /dashboards   →   { kind: "dashboard" } (no actionable context)
  4. Launcher shows:
    • Context banner at top: "Context: FGVM01TM24006844 (42024060)" with a "Clear context" link
    • Relevant tools surfaced first, each with a pre-filled state based on context (e.g. Port Scope loaded with the server already in the list)
    • Full tool catalog below, ungated — every plugin tool is always reachable from the launcher
  5. If no context detected: launcher opens to the catalog view with no banner (equivalent to opening the popup icon today)

Summary of this revision

Sidebar entry: "FM Toolkit" appended below Teams & Activity, styled to match FortiMonitor's native nav items, with a subtle plugin marker (blue dot).

Context-awareness lives in the launcher, not the entry. Clicking the entry passes the current path to the launcher; the launcher surfaces relevant tools first or shows the catalog. One stable entry in the nav, intelligent behavior behind it.

Scope clearly larger than Port Scope. The launcher becomes the single home for all plugin tools: Port Scope, Fabric Connection, Ask Claude, Attribute Management, Server Lookup, Template Management, etc. Adding new tools never means adding new nav entries.
