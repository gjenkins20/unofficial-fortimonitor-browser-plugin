FMN-150: in-page Columns menu placement

Two placement candidates for the toolkit's in-page hide/show menu on /report/ListServers. Popover content is identical between options; only the trigger differs.

Toolkit-injected UI is shown with the FM TK badge. Everything else mimics FortiMonitor's native page chrome. Both popovers persist while the operator clicks toggles; they close on outside-click. Reorder is intentionally absent (FMN-122 gate).

Option A: "Columns" button in the action bar (left of Clear Filters)

Option A pros. Lives inside the action bar the operator already uses (Add, Move, Delete, Tag, Download). Discoverable on first scan. Visually consistent with FortiMonitor's own button row. Caret signals a dropdown.

Option A cons. Adds horizontal weight to a row that's already busy on narrow viewports. Sits between Download and Clear Filters, so the operator's eye has to parse a fourth (toolkit) widget mid-row.
