Mockup notes (not rendered in final plugin):
· Structurally identical to the Remove tool's Step 1 — the Load step does not differ between Add and Remove. Only the title-bar name changes.
· This is Step 1 of 4. Next step is the group-level prompt (tool-add-port-scope_review.html).
· CSV input accepts either a plain list of server IDs (one per line) or a CSV with a server_id column. Additional columns are ignored.
· Before "Start review" is enabled, the plugin validates each ID against the current FortiMonitor session by silently reading its port scope in the background (concurrency cap 3). IDs with no matching device are flagged as warnings and skipped, not a hard fail.
· After "Start review", the plugin silently reads each device's port scope using the existing session, computes a fingerprint (sorted hash of {port_name, admin_status, oper_status} tuples), groups devices by identical fingerprint, then presents the interface-report prompt once per group. No visible tab navigation in the default mode — reads happen in the background.
· Session validity is shown passively — no OAuth or credential entry. Session is whatever the user already has in FortiMonitor.
· All reads and writes use the existing FortiMonitor session — the same one the WebGUI uses. Nothing runs outside the browser.
