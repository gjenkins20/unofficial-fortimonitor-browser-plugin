FMN-71 - Instances list: IP Address + DNS Name columns (augmentation)

Target: two columns inserted after Instance name in the /report/ListServers table: IP Address (numeric IPv4/IPv6) and DNS Name (everything else). Source is /report/get_idp_data?server_id={id} → pageData.instance.fqdn, classified client-side via regex. The opposite column shows an italic "not captured" placeholder. Both columns are sortable (client-side, current page only).

Design decisions and open items

  • Two columns, one source: fqdn is a single string. Classify with regex: matches /^(\d{1,3}\.){3}\d{1,3}$/ or /^[0-9a-f:]+$/i (IPv6) → IP column. Otherwise → DNS column. Exactly one populates per row; the other shows "not captured".
  • Placement: after Instance name, before Parent Group (columns 5-6 of 11). Name → IP → DNS scan order.
  • Not-captured style: italic, muted grey (#9AA4BC), text "not captured" with a tooltip disambiguating the cause (classifier mismatch vs. fetch failure).
  • Loading placeholder: pulsing skeleton bar, 90px (IP) or 140px (DNS).
  • Column widths: IP 130px (fits IPv6), DNS 200px with ellipsis overflow + full value in tooltip.
  • Sorting: click the column header to sort ascending, click again to flip descending, click a third time to clear. Active-sort header turns blue with an oriented chevron. IP column uses octet-aware numeric comparison (10.0.0.2 before 10.0.0.10); DNS uses case-insensitive string compare. Rows still loading and "not captured" rows sort to the bottom in both directions, so you only see "real" values at the top.
  • Sort scope is the current page (25 rows). FortiMonitor paginates server-side and the server doesn't know IPs, so our sort only reorders what's currently rendered. Changing page / applying a FortiMonitor sort resets our sort. Documented in a tooltip on the header.
  • Fetch strategy: per-row on render, concurrency 3 (matches scanner.js). In-memory cache keyed by server_id so pagination doesn't re-fetch devices you've already seen this session.
  • Sticky header: FortiMonitor's table renders a duplicate <thead> for the floating header. Both need the two columns, and clicking either must trigger the same sort.
